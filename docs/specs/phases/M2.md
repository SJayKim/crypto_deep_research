# M2 — One worker over A2A

> Phase 3 of 6 · Epic index: [`../M0-M5-vertical-slice-epic.md`](../M0-M5-vertical-slice-epic.md)
> Prev → [M1 — One worker, direct](./M1.md) · Next → [M3 — Fan-out + synthesizer + long-term READ](./M3.md)

## Inherits (from: [M1](./M1.md))

- **`workers/market/agent.py`** — the proven `data → reason → distill` market-worker.
  M2 wraps it; the agent logic is unchanged.
- **`mcp_server/`** — the running 4-tool HTTP server + BTC fixtures.
- **`contracts/a2a.py`** (from M0) — `JsonRpcRequest/Response`, `TaskParams`,
  `AgentCard`. M2 is the first phase to put these on the wire.

**Preconditions:** M1 acceptance criteria all pass (market-worker returns a valid
artifact when called directly; MCP-down yields `status="failed"`).

## Scope

Wrap `market-worker` as an A2A JSON-RPC 2.0 service (A1) with a static Agent Card at
`/.well-known/agent.json`; the orchestrator `dispatch` node calls it over A2A. First
over-the-wire vertical slice. Hand-rolled wire only (A1 already settled the SDK question).

## Acceptance criteria

1. `market-worker/service.py` serves `POST` JSON-RPC `analyze` and `GET /.well-known/agent.json`.
2. Orchestrator dispatches an A2A `analyze` task and receives a `WorkerArtifact`
   back; request/response validate against `contracts/a2a.py`.
3. E2E test runs the worker as an in-process ASGI app on a loopback port (T8) and
   asserts a real JSON-RPC round-trip (not an in-proc function call).
4. A malformed JSON-RPC request returns a structured `JsonRpcError`, not a 500 stack trace.

## Files

`workers/market/service.py`, `orchestrator/dispatch.py` (single-worker form),
`orchestrator/app.py` (minimal plan→dispatch→return), `tests/test_a2a_market.py`.

## Effort

A2A server + Agent Card ~3h, dispatch client ~2h, E2E ~2h.

## Applies (locked decisions)

A1, T8, context isolation (orchestrator receives only the artifact).

## Hands off (to: [M3](./M3.md))

- **`workers/market/service.py`** — the A2A service shape that workers 2–4 copy. After
  the 2nd worker exists, M3 extracts the shared parts into `workers/base.py` (C6).
- **`orchestrator/dispatch.py`** (single-worker form) — M3 generalizes this into an
  `asyncio.gather` fan-out over the worker set (P9, **never** `Send`).
- **`orchestrator/app.py`** (plan→dispatch→return) — M3 inserts `planner` (long-term
  READ) and `synthesize` nodes into this graph.

**Seam:** the A2A boundary is proven end to end for one worker. M3 multiplies the worker
and adds parallelism on top of a transport that is already known-good.
