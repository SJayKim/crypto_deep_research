# M1 — One worker, direct

> Phase 2 of 6 · Epic index: [`../M0-M5-vertical-slice-epic.md`](../M0-M5-vertical-slice-epic.md)
> Prev → [M0 — Contracts & scaffold](./M0.md) · Next → [M2 — One worker over A2A](./M2.md)

## Inherits (from: [M0](./M0.md))

- **`contracts/` package** — `WorkerArtifact` + validators (artifact emitted here must
  pass them), `mcp_tools` schemas (MCP server's I/O shapes).
- **`wiring.py`** — MCP URL read from env.
- **`sources/fixtures/`** — empty dir, ready for BTC fixtures.

**Preconditions:** M0 acceptance criteria all pass (contracts import; validators reject
6 key_points / over-length headline + key_point; `wiring.py` errors on a missing URL).

## Scope

MCP server (streamable HTTP) exposing all 4 tools backed by `FixtureSource`;
`market-worker` as a LangGraph agent (`data → reason → distill`) called **directly**
(no A2A); BTC fixtures for the 4 tools. Market-worker uses LLM reasoning (DESIGN
per-worker decision); the distill node emits a valid bounded `WorkerArtifact`.

## Acceptance criteria

1. MCP server responds to all 4 tool calls over streamable HTTP with the schemas in
   `contracts/mcp_tools.py`; tools are stateless and read-only (4 concurrent calls
   return identical data).
2. Calling `market-worker` directly with `"BTC"` pulls OHLCV via MCP, reasons, and
   returns a `WorkerArtifact(dimension="market", status="ok")` that passes all
   validators.
3. Worker-behavior test runs against **real Anthropic** and asserts the artifact is
   non-trivial (headline non-empty, ≥1 key point, ≥1 evidence) (T7).
4. MCP-down test uses a **deterministic stub**: MCP unreachable → worker returns
   `status="failed"`, never raises into the caller (T7b, A3 shape).

## Applies (locked decisions)

A2 (distill bounds), MCP boundary, T7/T7b.

## Hands off (to: [M2](./M2.md))

- **`workers/market/agent.py`** — a working `data → reason → distill` agent returning a
  valid `WorkerArtifact`. M2 wraps this exact agent behind an A2A service; the agent
  logic does not change.
- **`mcp_server/`** — the 4-tool HTTP server stays as-is through M2–M4 (only M5 swaps a
  source). Workers reach it as MCP clients.
- **BTC fixtures** — the deterministic data M2's wire test runs against.

**Seam:** the reasoning loop is proven in isolation. M2 adds the A2A transport around an
already-correct worker, so a wire failure can only be the protocol, not the logic.
